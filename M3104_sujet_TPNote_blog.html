Patron de développement MVC

On vous fournit une application de blog minimaliste (voire simpliste), qui permet d’afficher des posts (ou billets). Elle fonctionne et contient déjà deux posts (testez-la). Vous devrez la modifier et la compléter.
Vous devrez comprendre globalement le code et sa structure générale, le modifier et le compléter.
Pour vous aider vous pouvez dessiner un graphe des include et un graphe de parcours du site.
Pensez à faire des sauvegardes intermédiaires à chaque modification majeure.

Conseil : attention, comme vous l’avez vu en cours, mais comme vous ne l’avez pas vu en TP, il y a un contrôleur frontal (qui appelle des sous-contrôleurs dédiés).

Toutes les modifications devront respecter le patron de conception MVC tel qu’enseigné en cours et TP.

Améliorations

Avant de passer à la suite, constatez que le code est inutilement compliqué, simplifiez et améliorez-le en documentant (au fur et à mesure) dans la fiche de rendu ce que vous faites et pourquoi vous le faites. Ne négligez pas cette étape, vous vous faciliterez la vie pour la suite. C’est du temps investi, pas du temps perdu.

Saisie d’un post

Le site fourni permet d’afficher des posts, mais pas de les saisir. Vous allez devoir le compléter pour effectuer ces saisies.

Ajout d’un lien

Ajoutez le choix Saisie dans le menu sous forme d’un lien.

Page de saisie

Créez une page (statique) accessible via le lien précédent qui permet la saisie via un formulaire HTML. Le code HTML pourra ressembler à ce qui suit (à compléter) :

<p>Veuillez saisir un nouveau post :</p>
<form action="" method="GET">
<fieldset>
Auteur : <input type="texte" name="À Compléter"><br />
Contenu : <input type="texte" name="À Compléter">
<input type="hidden" name="action" value="À Compléter">
<input type="hidden" name="controller" value="À Compléter">
<input type="submit" value="Enregistrer">
</fieldset>
</form>

Enregistrement d’un post

Créez une page (dynamique) accessible via le formulaire précédant. Cette page devra afficher un message de réussite (ou d’erreur).

Affichage du post enregistré

La page précédante, en plus du message de réussite, devra afficher le post enregistré.

Ajout de fonctions

Proposez d’autres améliorations (connexions par utilisateurs, mémos privés non visibles sauf par l’utilisateur, etc.) toutes documentées dans la fiche de rendu

Gestion des droits

La gestion des droits lors des TPs n’était pas satisfaisante d’un point de vue sécurité, et ne permetait pas le développement à plusieurs. On vous propose un nouveau script : setup-public-group qui va accorder les droits en écriture au seul groupe, groupe qui contiendra les membres du binôme, Apache, et votre enseignant. Vous devez déclarer votre binôme à votre enseignant pour qu’il créé votre groupe (au sens *nix).

Une fois le travail terminé

Déposez la fiche de rendu remplie sur Chamilo. Ne modifiez plus votre code après la date limite, vous n’avez pas besoin de le rendre, votre enseignant viendra le consulter sur votre compte.
